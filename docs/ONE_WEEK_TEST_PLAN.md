# One-Week Journalist Test Plan

## Test journalist

**Anika Patel**  
Website: `www.whatsoninmzansi.co.za`  
Sections: Food & Weekend Markets, Family & Kids Days Out

## Daily test rhythm

### Day 1 — Setup and baseline

- Create 2 sandbox articles in mock mode.
- Confirm artifacts are saved.
- Confirm Telegram preview works.
- Reject one article and approve one article.
- Check memory candidates.

### Day 2 — Tone test

- Give Anika a specific topic.
- Check if tone feels warm but not childish.
- Give feedback on words/phrases that feel AI-generated.
- Approve memory candidates only if useful.

### Day 3 — Image test

- Generate 3 images or placeholders.
- Check prompts for AI slop risk.
- Regenerate image with feedback.
- Save image preference memories.

### Day 4 — Source discipline

- Use a topic with real venues/events.
- Confirm unverified details are flagged.
- Reject any article that invents dates, prices, or venue claims.

### Day 5 — Revision behavior

- Request revision on one article.
- Check if feedback is stored properly.
- Confirm memory candidates are useful.

### Day 6 — Editorial consistency

- Create multiple articles in the same section.
- Check if structure improves.
- Check whether approved memories influence style.

### Day 7 — Scale decision

Decide whether to:

- keep Anika and improve her,
- add Zola as the second test journalist,
- or start connecting WordPress publishing adapter.

## Pass criteria

- No article publishes publicly.
- No invented sources pass unnoticed.
- Image prompts avoid slop.
- Telegram approval works.
- Memory candidates are helpful and controllable.
- Dashboard gives enough visibility to trust the system.
